## 📋 Deployment Steps

### Step 1: Update Supabase Database

Run this SQL in your Supabase SQL Editor:

```sql
-- Create short_urls table
CREATE TABLE IF NOT EXISTS short_urls (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id UUID NOT NULL,
    original_url TEXT NOT NULL,
    alias TEXT UNIQUE NOT NULL,
    short_url TEXT NOT NULL,
    title TEXT NOT NULL DEFAULT 'Untitled',
    clicks INTEGER DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Enable Row Level Security
ALTER TABLE short_urls ENABLE ROW LEVEL SECURITY;
```

### Step 2: Deploy to Cloudflare

Since you're already using Cloudflare Pages, you just need to push your changes:

```bash
# Stage all changes
git add .

# Commit the changes
git commit -m "Add URL Shortener feature"

# Push to GitHub (Cloudflare will auto-deploy)
git push origin main
```

Cloudflare Pages will automatically detect the changes and redeploy your site!

## 🔗 New API Endpoints Added

### Protected Endpoints (Require Authentication):

#### 1. Get All Short URLs
```
GET /short-urls
Headers: Authorization: Bearer <token>
Response: Array of short URL objects
```

#### 2. Create Short URL
```
POST /short-urls
Headers: Authorization: Bearer <token>
Body: {
  "original_url": "https://example.com/long/url",
  "alias": "my-link",
  "title": "My Link"
}
Response: {
  "message": "Created",
  "short_url": "https://xqpl-tool.pages.dev/s/my-link",
  "id": 123
}
```

#### 3. Delete Short URL
```
DELETE /short-urls/<id>
Headers: Authorization: Bearer <token>
Response: {"message": "Deleted"}
```

### Public Endpoint (No Authentication):

#### 4. Redirect Short URL
```
GET /s/<alias>
Response: 302 Redirect to original URL
```

This endpoint:
- ✅ Works without authentication
- ✅ Increments click counter
- ✅ Redirects to the original URL

## 🎯 How It Works

### Creating a Short URL:
1. User enters a long URL in the URL Shortener page
2. Optionally adds a custom alias (or auto-generates one)
3. Frontend sends POST request to `/short-urls`
4. Worker creates entry in Supabase `short_urls` table
5. Returns the short URL: `https://xqpl-tool.pages.dev/s/{alias}`

### Using a Short URL:
1. Someone visits `https://xqpl-tool.pages.dev/s/my-link`
2. Worker looks up the alias in the database
3. Increments the click counter
4. Redirects to the original URL

## 🐛 Troubleshooting

### Issue: "Table short_urls does not exist"
**Solution**: Run the SQL migration in Supabase (Step 1 above)

### Issue: "Alias already exists"
**Solution**: Choose a different alias or leave it empty for auto-generation

### Issue: Short URLs not working after deployment
**Solution**: 
1. Check Cloudflare Pages deployment logs
2. Verify the `worker.py` file was deployed
3. Check Supabase connection in environment variables

### Issue: Redirects not working
**Solution**: 
1. Make sure the `/s/{alias}` endpoint is before the auth check in `worker.py`
2. Verify the alias exists in the database
3. Check that the original URL includes `http://` or `https://`
