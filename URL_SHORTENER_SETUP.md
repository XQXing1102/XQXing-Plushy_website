## 📋 Database Setup

### Step 1: Run the SQL Migration

You need to add the `short_urls` table to your Supabase database.

**Option A: Using Supabase Dashboard**
1. Go to your Supabase project dashboard
2. Navigate to **SQL Editor**
3. Run this SQL command:

```sql
-- Short URLs table
CREATE TABLE IF NOT EXISTS short_urls (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id UUID NOT NULL,
    original_url TEXT NOT NULL,
    alias TEXT UNIQUE NOT NULL,
    short_url TEXT NOT NULL,
    title TEXT NOT NULL DEFAULT 'Untitled',
    clicks INTEGER DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Enable Row Level Security
ALTER TABLE short_urls ENABLE ROW LEVEL SECURITY;
```

**Option B: Re-run the full setup**
If you prefer to recreate all tables, run the updated `supabase_setup.sql` file which now includes the short_urls table.

## 🚀 How to Use

### 1. Start the Backend Server
```bash
cd backend
python app.py
```
The server will run on `http://127.0.0.1:9999`

### 2. Access the URL Shortener
- Login to your account
- Click on the **🔗 URL Shortener** tile from the dashboard
- Or navigate directly to: `http://localhost:5500/url-shortener.html`

### 3. Create Short URLs
1. Enter the original long URL
2. (Optional) Add a custom alias (e.g., "my-link")
3. (Optional) Add a title for easy identification
4. Click **Create Short URL**

### 4. Features Available
- **Copy URL**: One-click copy to clipboard
- **Visit URL**: Open the short URL in a new tab
- **QR Code**: Generate and download QR code
- **Analytics**: View click count for each URL
- **Search**: Find URLs by title, original URL, or alias
- **Sort**: By recent, most clicks, or alphabetically

## 🔧 API Endpoints Added

### Get all short URLs
```
GET /short-urls
Headers: Authorization: Bearer <token>
```

### Create short URL
```
POST /short-urls
Headers: Authorization: Bearer <token>
Body: {
  "original_url": "https://example.com/very/long/url",
  "alias": "my-link",
  "title": "My Important Link"
}
```

### Delete short URL
```
DELETE /short-urls/<id>
Headers: Authorization: Bearer <token>
```

### Redirect (public endpoint)
```
GET /s/<alias>
```
This endpoint redirects to the original URL and increments the click counter.

## 🐛 Troubleshooting

### Issue: "Alias already exists"
**Solution**: Choose a different alias or leave it empty for auto-generation

### Issue: "Network error"
**Solution**: Make sure the backend server is running on port 9999

### Issue: Short URLs not loading
**Solution**: 
1. Check if the `short_urls` table exists in Supabase
2. Verify your Supabase credentials in `backend/.env`
3. Check browser console for errors

### Issue: Redirect not working
**Solution**: Make sure you're using the full URL including the protocol (http:// or https://)
